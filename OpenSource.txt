Descobrindo e avaliando projeto
- o Github é uma rede social para desenvolvedores
- podemos procurar por projetos, rankear com estrelas projetos de interesse, seguir perfis etc

Issues
- tarefas/problemas a serem resolvidos no projeto
- ao corrigirmos um erro apontado por uma issue, podemos apagar a issue via commit
    - cada issue tem uma numeração #<Num>
    - ao dar commit corrigindo o arquivo, finalizamos a mensagem com Closes #<Num>

Fork
- ao darmos fork em um projeto de algum usuário no Github, criamos no nosso perfil um repositório novo
com o conteúdo daquele projeto. 
- toda e qualquer modificação feita no projeto copiado se restringe à cópia no nosso repositório
- podemos deletar um repositório em settings, ao fim da página

Realizando Pull Request
- através do pull request nós solicitamos que as alterações feitas nos projetos copiados via fork
sejam incorporadas ao projeto original
- o proprietário do projeto original decide se aceita ou não o pull request
- a numeração da pull request continua a contagem de índice de issues ( caso haja 2 issues, a primeira
pull request seria #3 )

Verificando e aceitando um pull request
- quando recebemos um pull request, devemos criar um branch para testar se as modificações sugeridas
são pertinentes. Estando tudo ok, podemos fazer o merge pelo próprio Github
